## Pass 9 — `+experiment/` + `+kinematics/` + `+kinetics/` + `+emg/`

Codex found 3 issues — all fixed:

### 1. `+experiment/extractFeatures.m` — global summary copied to every stride
> Motion/GRF summaries are global means copied to every stride, not per-stride features.

**Fix**: per-stride GRF features now index the k-th stance phase (`grfFeats.peak_vertical_N(k)`); motion features fall back to the side-specific TRIAL summary (honest about what the toolbox can compute today). `extractFeatures` doc block updated.

### 2. `+kinematics/computeJointAngles.m` — R/L summary collision
> Summary fields overwrite R with L and are whole-trial, not per-stride.

**Fix**: summaries are now keyed by side (`knee_peak_flex_R`, `knee_peak_flex_L`, etc.). `extractFeatures` reads the side-specific field per row.

### 3. `+kinetics/grfFeatures.m` — off-by-one in stance duration filter
> `(stops - starts) >= minDur` drops contacts of exactly 100 ms because indices count differences between samples, not sample counts.

**Fix**: changed to `(stops - starts + 1) >= minDurSamples` (sample-count semantics).

## Pass 10 — Final adversarial sweep + Copilot friendliness

### Critical findings (3 — all fixed)

#### 1. `compareConditions.m` ignored `decisionTree` non-parametric recommendation

> For K >= 3 within-subject data it always runs RM-ANOVA, even when `decisionTree` would recommend Friedman. This contradicts the pipeline docs and can produce paper-wrong p-values.

**Fix**: `compareConditions.m` now checks `decision.normality_ok` and routes to:
- 2 within + non-parametric → `signrank` (Wilcoxon)
- 2 between + non-parametric → `ranksum` (Mann-Whitney U)
- 3+ within + non-parametric → `friedman`
- 3+ between + non-parametric → `kruskalwallis` + BH-FDR post-hoc

Effect sizes adapted (`r = Z / sqrt(N)` for Wilcoxon, etc.).

#### 2. `generateMethods.m` fabricated acquisition descriptions

> Always says motion/GRF/EMG/loadcell were recorded and processed, regardless of what exists in `cmp`. Says "Three modalities" while listing four bullets. Unsafe to paste into a paper.

**Fix**: `generateMethods.m` now scans `cmp` field names to detect actually-present modalities (`has.robot`, `has.motion`, `has.grf`, `has.emg`, `has.loadcell`) and emits text only for those. Modality count is computed from the booleans.

#### 3. `MULTIMODAL_PIPELINE.md` documented unimplemented APIs

> `syncModalities`, `detectStrides`, `normalizeStrides`, `ensembleAverage`, `loadGroup`, `compareGroup`, `makeAllPaperFigures` are documented but absent. `generateMethods(groupCmp, '~/...')` is wrong; implementation requires `'OutFile', path`.

**Fix**: docs rewritten to reflect actual API. Removed unimplemented function calls; added "Roadmap:" notes for future wrappers; corrected `generateMethods` call signature; replaced `makeAllPaperFigures` section with direct `exportAllJournals` examples; replaced `loadGroup` / `compareGroup` with explicit subject-loop pattern.

### Copilot friendliness — score 7/10

> The prompt comments are concrete enough for broad matching, especially examples 01, 02, 03, 07. Weakness: several prompts do not name the exact `hwalker.*` function or required input shape.

**Fix**: added a `CANONICAL Copilot prompt:` block to all 7 example scripts, each containing the exact function chain to call. This gives MATLAB Copilot an unambiguous target when the user types a natural-language question.
